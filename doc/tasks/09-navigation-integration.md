# 导航与页面联调模块任务

目标：串联五大页面、底部 Tab、跨页面状态同步和基础空状态。

## 最小任务清单

- [ ] 确认当前 App 导航方案。
- [ ] 建立五大页面路由：首页、食谱计划、数据看板、运动记录、我的。
- [ ] 将底部 Tab 接入主 App。
- [ ] 设置首页为默认首屏。
- [ ] 确保页面切换不清空业务状态。
- [ ] 首页体重记录后，同步数据看板累计减重和体重趋势。
- [ ] 首页饮食打卡后，同步食谱计划页饮食状态和看板完成率。
- [ ] 运动页记录后，同步首页运动状态和看板运动状态。
- [ ] 个人中心修改初始体重后，同步首页和看板减重进度。
- [ ] 个人中心切换计划后，同步首页今日食谱和食谱计划选中计划。
- [ ] 补打卡后，同步首页、看板和历史记录。
- [ ] 首次使用时展示体重设置空状态。
- [ ] 无体重记录时展示看板空状态。
- [ ] 无运动记录时展示运动历史空状态。
- [ ] 无历史打卡时展示历史空状态。
- [ ] 处理 AsyncStorage 读取中的加载状态。
- [ ] 处理本地数据读取失败降级状态。
- [ ] 确保所有长文本在 iOS 和 Android 小屏不溢出。
- [ ] 确保底部 Tab 不遮挡最后一个卡片。

## 验收标准

- [ ] 五大页面均可通过底部 Tab 进入。
- [ ] 首页是 App 首屏。
- [ ] 跨页面状态同步无明显延迟或错乱。
- [ ] 空状态覆盖首次使用和无记录场景。
- [ ] 页面在小屏设备可滚动且不遮挡内容。
