# 数据模型与静态食谱模块任务

目标：建立全局 TypeScript 类型、静态食谱数据和计划计算基础能力，保证后续页面和状态模块使用同一套数据口径。

## 最小任务清单

- [ ] 定义 `PlanType`、`CheckInStatus`、`MealType`、`RecipePhase` 等基础枚举类型。
- [ ] 定义 `RecipeMeal`、`RecipeDay`、`WeightRecord`、`WeightSettings` 类型。
- [ ] 定义 `DietCheckIn`、`FoodViolation`、`DailyCheckIn` 类型。
- [ ] 定义 `ExerciseRecord`、`PlanProgress`、`UserSettings` 类型。
- [ ] 建立秦昊15天静态食谱数据文件，完整录入 Day 1 到 Day 15。
- [ ] 建立大冰7天循环静态食谱数据文件，完整录入循环第1天到第7天。
- [ ] 为秦昊食谱写入原文每日预计掉秤，总目标固定为 11.3 斤。
- [ ] 为大冰食谱写入非数值型预估文案，不自行估算每日掉秤。
- [ ] 确保所有静态食谱 `sourceLocked` 为 `true`，不提供编辑字段。
- [ ] 实现根据计划类型和天数读取指定食谱的纯函数。
- [ ] 实现大冰循环日计算：`((executionDay - 1) % 7) + 1`。
- [ ] 实现计划执行天数计算，使用本地自然日。
- [ ] 补齐空值处理：未找到食谱时返回可控空状态，不让页面崩溃。
- [ ] 验证食谱内容未被摘要、改写或新增未确认餐单。

## 验收标准

- [ ] 所有业务模块可以从统一类型文件导入类型。
- [ ] 秦昊15天和大冰7天食谱均可按天读取。
- [ ] 大冰第8天能正确回到循环第1天。
- [ ] 大冰计划不出现自造掉秤数字。
- [ ] 静态数据不写入 AsyncStorage。
