# 首页打卡模块任务

目标：实现每日执行入口，展示当前计划、今日食谱、打卡状态、连续打卡和真实减重进度。

## 最小任务清单

- [ ] 建立首页容器页面。
- [ ] 接入当前计划和计划开始日期。
- [ ] 实现 `TodayPlanHeaderCard`。
- [ ] 展示当前计划标签。
- [ ] 展示本地自然日日期。
- [ ] 展示当前执行第几天。
- [ ] 秦昊计划展示原文当日预计掉秤。
- [ ] 大冰计划展示“稳步减脂、维稳塑形”类文案，不展示数字。
- [ ] 秦昊计划超过15天时展示切换大冰提示。
- [ ] 实现 `TodayCheckInSummaryCard`。
- [ ] 展示体重、饮食、运动三项完成状态。
- [ ] 任意一项完成时展示当日打卡成功。
- [ ] 三项完成时展示完整打卡。
- [ ] 实现体重记录入口。
- [ ] 保存体重记录后同步每日打卡体重状态。
- [ ] 实现饮食打卡入口。
- [ ] 支持选择按食谱完成或未按食谱完成。
- [ ] 支持填写违规进食记录入口。
- [ ] 保存饮食打卡后同步每日打卡饮食状态。
- [ ] 实现运动打卡入口跳转或快捷入口。
- [ ] 保存运动记录后同步每日打卡运动状态。
- [ ] 实现 `StreakCard`。
- [ ] 展示连续打卡天数。
- [ ] 补打卡后能重新显示正确连续天数。
- [ ] 实现 `TodayRecipePreviewCard`。
- [ ] 展示早餐、午餐、晚餐和原文提供的加餐。
- [ ] 展示当天核心规则。
- [ ] 展示禁忌或注意事项。
- [ ] 不摘要替代原始餐单。
- [ ] 实现 `WeightProgressCard`。
- [ ] 使用初始体重减最新体重计算真实累计减重。
- [ ] 秦昊15天进度目标固定为 11.3 斤。
- [ ] 未填写初始体重或无体重记录时展示空状态。
- [ ] 页面接入底部 Tab。

## 验收标准

- [ ] 首页能作为 App 首屏使用。
- [ ] 今日食谱根据当前计划和日期正确变化。
- [ ] 任意一项打卡即可让当天进入成功状态。
- [ ] 三项都完成时显示完整打卡。
- [ ] 真实减重进度不使用预估掉秤累加。
