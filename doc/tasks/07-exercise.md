# 运动记录模块任务

目标：实现每日爬坡快走和力量训练记录、备注和运动历史查看。

## 最小任务清单

- [ ] 建立运动记录页面容器。
- [ ] 实现 `ExerciseDateHeader`。
- [ ] 默认展示本地当天日期。
- [ ] 支持选择历史日期用于补记录。
- [ ] 禁止选择未来日期。
- [ ] 实现爬坡快走 `ExerciseCheckCard`。
- [ ] 默认目标时长为60分钟。
- [ ] 支持勾选完成状态。
- [ ] 支持输入实际时长。
- [ ] 实现力量训练 `ExerciseCheckCard`。
- [ ] 默认目标时长为30分钟。
- [ ] 支持勾选完成状态。
- [ ] 支持输入实际时长。
- [ ] 实现运动记录保存逻辑。
- [ ] 任意运动项填写或完成即标记为有运动记录。
- [ ] 两项都完成时标记为运动完整完成。
- [ ] 保存后同步每日打卡运动状态。
- [ ] 实现 `ExerciseNoteCard`。
- [ ] 支持保存当天运动备注。
- [ ] 实现 `ExerciseTipsCard`。
- [ ] 展示爬坡注意事项：大基数低速、小坡度、无跑跳、保护膝盖。
- [ ] 展示力量训练建议：臀桥、深蹲、平板支撑、手臂塑形、无负重。
- [ ] 展示运动核心作用说明。
- [ ] 实现 `ExerciseHistoryList`。
- [ ] 展示日期、爬坡状态、爬坡时长。
- [ ] 展示力量状态、力量时长。
- [ ] 展示备注。
- [ ] 无运动记录时展示空状态。
- [ ] 不自动估算或展示 kcal 消耗。

## 验收标准

- [ ] 爬坡和力量训练可以分别记录。
- [ ] 任意一项记录会触发当天 partial 打卡。
- [ ] 两项完成才计为运动完整完成。
- [ ] 历史运动记录重启后不丢失。
- [ ] 页面不展示自造运动消耗热量。
