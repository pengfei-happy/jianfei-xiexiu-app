# 质量验证与验收模块任务

目标：按需求文档和详细设计验证第一版实现，确保核心规则、离线存储和展示口径正确。

## 最小任务清单

- [ ] 验证 App 可离线打开。
- [ ] 验证首次进入默认秦昊15天计划。
- [ ] 验证用户可填写初始体重和目标体重。
- [ ] 验证体重单位固定为斤。
- [ ] 验证秦昊15天完整餐单可查看。
- [ ] 验证秦昊每日预计掉秤来自原文。
- [ ] 验证秦昊总预估目标为 11.3 斤。
- [ ] 验证大冰7天循环完整餐单可查看。
- [ ] 验证大冰第8天回到循环第1天。
- [ ] 验证大冰不展示自造每日掉秤数字。
- [ ] 验证首页能展示当前计划、日期、今日食谱和打卡状态。
- [ ] 验证完成体重记录可让当天打卡成功。
- [ ] 验证完成饮食打卡可让当天打卡成功。
- [ ] 验证完成运动打卡可让当天打卡成功。
- [ ] 验证三项都完成时当天为完整打卡。
- [ ] 验证自然日断卡后连续打卡天数清零。
- [ ] 验证补打卡会重新计算连续打卡。
- [ ] 验证未按食谱完成仍算饮食记录已填写。
- [ ] 验证未按食谱完成不计入饮食完成率。
- [ ] 验证体重趋势支持周视图。
- [ ] 验证体重趋势支持月视图。
- [ ] 验证同一天多次体重记录取最后一次作为图表值。
- [ ] 验证运动记录支持爬坡和力量分别打卡。
- [ ] 验证任意运动项填写即可产生运动记录。
- [ ] 验证两项运动都完成才是运动完整完成。
- [ ] 验证历史记录可在 App 内查看。
- [ ] 验证计划切换不删除历史记录。
- [ ] 验证重启 App 后本地数据不丢失。
- [ ] 验证重置打卡和体重记录需要二次确认。
- [ ] 验证重置全部本地数据需要二次确认。
- [ ] 验证热量摄入、运动消耗、热量缺口不被自动计算。
- [ ] 验证长食谱文案不溢出卡片。
- [ ] 验证底部 Tab 不遮挡页面最后内容。
- [ ] 在 iOS 模拟器或真机完成基础冒烟测试。
- [ ] 在 Android 模拟器或真机完成基础冒烟测试。

## 验收标准

- [ ] 所有核心业务规则与 `doc/proposal.md` 一致。
- [ ] 所有设计约束与 `doc/detailed-design.md` 一致。
- [ ] 无联网依赖阻塞食谱查看和历史数据查看。
- [ ] 无编造食谱、热量或掉秤数据。
- [ ] 冒烟测试通过后再标记各模块完成。
