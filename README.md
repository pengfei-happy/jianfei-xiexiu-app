# 减肥饮食打卡 App

一个面向手机端使用的减肥/饮食打卡 Web App，围绕「今日食谱、体重记录、运动记录、数据看板」做本地化记录。当前版本已经部署到 Netlify，可以直接通过浏览器访问。

## 在线体验

- 公开访问地址：[https://jianfei-xiexiu-app.netlify.app](https://jianfei-xiexiu-app.netlify.app)
- GitHub 仓库：[https://github.com/pengfei-happy/jianfei-xiexiu-app](https://github.com/pengfei-happy/jianfei-xiexiu-app)

> 说明：当前数据保存在用户自己的浏览器本地存储里，不是服务器数据库。换设备、换浏览器或清理浏览器数据后，记录不会自动同步。

## 这个 App 解决什么问题

减肥计划最难的不是知道吃什么，而是每天能不能持续记录、及时看到状态，并且不要被复杂表单劝退。

这个 App 的目标是把每天的核心动作压缩成几个明确步骤：

1. 看今天对应哪一天食谱。
2. 记录体重、饮食、运动。
3. 查看当天是否完成，以及连续打卡情况。
4. 在数据看板里回看体重和完成趋势。

## 核心功能

### 首页

- 展示今日计划和当前食谱。
- 展示今日进度：体重、饮食、运动三项完成情况。
- 支持快速保存体重、饮食打卡和运动记录。
- 如果在饮食页选择了某一天食谱并打卡，首页会同步显示对应食谱。

### 饮食 & 食谱

- 支持秦昊 15 天计划。
- 支持大冰 7 天循环计划。
- 每天食谱带有对应视觉标识，例如豆浆/牛奶日、玉米日、水果日、蛋白日、蔬菜日等。
- 可以按食谱完成，也可以记录未按食谱。

### 运动记录

- 支持爬坡快走计时与保存。
- 支持徒手力量训练计时与保存。
- 运动记录会同步影响当天打卡状态。

### 数据看板

- 展示体重记录与减重进度。
- 展示饮食、运动和总体打卡完成情况。
- 用真实记录计算，不做服务器端数据同步。

### 我的

- 支持设置初始体重和目标体重。
- 支持切换当前计划。
- 支持清理本地记录。

## 数据逻辑

当前版本是 local-first，本地优先：

- 体重记录、饮食打卡、运动记录都保存在浏览器本地。
- 每个日期会生成当天的打卡状态。
- 当天任意一项记录完成后，会视为当天已有记录。
- 三项都完成时，当天状态为完整打卡。
- 饮食页选择某一天食谱并打卡后，首页会优先展示该日期保存的食谱，而不是只按自然天数推算。

## 本地运行

```bash
npm install
npm run web -- --port 8081
```

本地预览地址通常是：

```text
http://localhost:8081
```

## 构建与部署

项目使用 Netlify 部署，配置文件是 `netlify.toml`。

```toml
[build]
  command = "npm run build:web"
  publish = "dist"
```

本地构建命令：

```bash
npm run build:web
```

生产部署后访问：

```text
https://jianfei-xiexiu-app.netlify.app
```

## 当前限制

- 暂时没有账号系统。
- 暂时没有服务器数据库。
- 数据不会跨设备自动同步。
- 清理浏览器本地数据后，历史记录会丢失。
- 食谱内容是静态计划数据，不做在线动态更新。
